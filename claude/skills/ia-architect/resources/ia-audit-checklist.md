# IA監査チェックリスト（Fail / Risk / OK）

## 1) IAとナビの混同
- [ ] IA（分類・階層・メタデータ・関係）を、UI（メニュー/ヘッダ）抜きで説明できる → OK / Risk / Fail
- [ ] サイトマップが「可視化ツール」であり、IAそのものと混同されていない → OK / Risk / Fail
  参考: https://www.nngroup.com/articles/information-architecture-sitemaps/

## 2) タスク中心の分類
- [ ] トップ分類が部署/組織図ではなく、ユーザーの用事（タスク）で説明される → OK / Risk / Fail
- [ ] よくある到達経路（Top tasks）が最短化されている → OK / Risk / Fail

## 3) 命名の一貫性
- [ ] 用語集（ラベル辞書）が存在し、統制されている → OK / Risk / Fail
- [ ] 同義語・表記揺れ・略語のルールがある → OK / Risk / Fail

## 4) スケール
- [ ] 増加前提で「階層 + メタデータ」による設計になっている → OK / Risk / Fail
- [ ] "その他"カテゴリに流入し続けない構造 → OK / Risk / Fail

## 5) Findability（検索・絞り込み）
- [ ] 検索対象フィールド、同義語、ゼロ件救済が定義されている → OK / Risk / Fail
- [ ] ファセット（絞り込み）で主要タスクが高速化される → OK / Risk / Fail
  参考: https://baymard.com/learn/information-architecture-ux

## 6) 検証（カードソート / ツリーテスト）
- [ ] カードソート計画（対象カード、参加者、分析方法）がある → OK / Risk / Fail
  参考: https://www.nngroup.com/articles/card-sorting-definition/
- [ ] ツリーテスト計画（タスク、成功条件、分析）がある → OK / Risk / Fail
  参考: https://www.optimalworkshop.com/101-guides/information-architecture-101/tree-testing-and-information-architecture

## 7) 運用（ガバナンス）
- [ ] 追加/変更の基準、レビュー責任、変更管理がある → OK / Risk / Fail
- [ ] 計測（検索語、ゼロ件率、到達率など）で継続改善できる → OK / Risk / Fail
