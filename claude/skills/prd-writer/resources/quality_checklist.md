# Quality Checklist（PRDレビュー用）

## Decision-ready（意思決定可能か）
- [ ] Problem / target customer が1〜3行で言える
- [ ] Why now が根拠（Evidence）とセットで書かれている
- [ ] Non-goals が明確で期待値が下げられている

## Metrics（測れるか）
- [ ] KPIが定義されている（何を/どう測る）
- [ ] ガードレール（悪化させない指標）がある
- [ ] baseline と measurement window がある

## Scope（作れるか）
- [ ] Appetite/Timebox が明示
- [ ] Must が絞られている（多すぎない）
- [ ] No-gos / Rabbit holes が書かれている

## Risks（落とし穴が潰れているか）
- [ ] Top risks に mitigation がある
- [ ] 依存（他チーム/外部/移行）が書かれている

## Open Questions（止まりどころがあるか）
- [ ] 未確定事項がOpen Questionsに集約されている
- [ ] Owner / Due が書かれている

## Optional（必要なら）
- [ ] NFR（性能/セキュリティ/運用）が妥当
- [ ] Rollout/Observability/Support が現実的
- [ ] PR/FAQで顧客価値の一貫性が確認できる
