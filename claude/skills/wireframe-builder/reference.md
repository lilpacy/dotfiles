# Wireframe Builder - Reference Guide

## ワイヤーフレームとは

画面ワイヤーフレームは、Web/アプリの**各画面の「構造」と「機能（何がどこにあり、どう使うか）」を、見た目の装飾（色・フォント・画像など）より前に整理して表す設計図**。

### 関連用語との違い

| 用語 | 主役 | 目的 |
|------|------|------|
| **ワイヤーフレーム** | レイアウト/情報構造/導線 | 構造と機能の合意 |
| **モックアップ** | 最終に近い見た目（ビジュアル） | デザインの確認 |
| **プロトタイプ** | 動きや操作感（インタラクション） | 操作性の検証 |

---

## なぜ必要か（価値）

1. **"何を作るか"の合意を早く・安く取れる**
   - 装飾前の段階で、画面構成・導線・情報の優先順位をチームで揃えやすい

2. **実装前に、要件漏れ・矛盾・例外（エッジケース）を炙り出せる**
   - フォームの入力制約、空状態、エラー、権限で見える/見えないなどを先に考えられる

3. **設計→実装の手戻りを減らす**
   - "見た目を作ってから構造が崩れる"事故を避けられる（順序として「骨組み→見た目」）

---

## フィデリティ（精度）の選択

### 低忠実度（Lo-Fi）
- **用途**：アイデア出し・議論・スピード重視
- **特徴**：手書きレベル、ラフなボックス
- **適する場面**：初期検討、ブレスト、方向性確認

### 中忠実度（Mid-Fi）
- **用途**：仕様検討・見積もり・レビュー
- **特徴**：コンポーネント粒度、状態も含む
- **適する場面**：開発前の仕様Fix、ステークホルダーレビュー

### 高忠実度（Hi-Fi）は注意
- 色・フォントに寄せると**議論が逸れやすい**
- ワイヤーフレームの段階では避けるべき

## 状態（States）の考慮

各画面・コンポーネントで以下の状態を検討する：

### Empty State（空状態）
- データがない場合の表示
- 次のアクションへの誘導

### Loading State（読み込み中）
- スケルトン or スピナー
- どこに表示するか

### Error State（エラー）
- エラーメッセージの位置
- リトライ方法

### Success State（成功）
- 完了メッセージ
- 次のステップへの誘導

### Permission State（権限）
- 権限がない場合の表示
- 表示しない vs 無効化 vs エラー

## よくある失敗と対策

| 失敗 | 対策 |
|------|------|
| 色やUI装飾に時間を使う | 「構造・導線・優先順位」の議論だと明示 |
| 画面単体は良いが遷移すると詰む | ユーザーフローで検証、動的ならWireflow |
| 「実装できない」「仕様足りない」 | 注釈（状態・制約・例外）を最初からセット |
| レスポンシブが考慮されていない | PC/SPの差分を明示 |
| 例外パターンが抜けている | エッジケースリストを作成 |

## Wireflow（画面＋フロー）

動的なプロセス（申請、購入、設定ウィザード等）は、ワイヤーフレーム単体だと伝わりにくい。
画面とフローを統合した**Wireflow**で表現する。

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│ Step 1  │ ──→ │ Step 2  │ ──→ │ Step 3  │
│ 入力    │     │ 確認    │     │ 完了    │
└─────────┘     └─────────┘     └─────────┘
     │               │
     ↓               ↓
  [戻る]          [修正]
```

## レスポンシブ対応

PC/SPで構造が変わる場合、両方のワイヤーを作成する。

### PC版
```
┌─────────────────────────────────────────┐
│ Header                                  │
├──────────┬──────────────────────────────┤
│ Sidebar  │ Main Content                 │
└──────────┴──────────────────────────────┘
```

### SP版
```
┌─────────────────┐
│ Header      [≡] │
├─────────────────┤
│ Main Content    │
│                 │
│                 │
└─────────────────┘

※ サイドバーはハンバーガーメニュー内に格納
```
