# Mockup Intake Questions(最小ヒアリング)

すでに分かっているものだけ回答でOK。分からないものは「不明」でOK(Assumptionsに回す)。

## 必須項目

1. **対象**:どの機能/プロジェクト?対象画面一覧は?(最低:画面名の列挙)

2. **目的**:合意形成?開発引き渡し?ブランド検証?投資家向け?

3. **デバイス**:PCのみ?SP必須?主要ブレークポイント(例:1440/1024/768/390)

## 既存資産

4. **デザイン基準**:ブランドカラー/フォント/コンポーネント/デザインシステム(あるなら参照先)

5. **既存ドキュメント**:画面遷移図/PRD/要件/ユーザーフロー(あるなら参照先)

## 内容

6. **状態**:最低限必要な状態(通常/空/ローディング/エラー/成功/無効)?

7. **コンテンツ**:実データ例(なければリアルなサンプルをこちらで作る)

8. **優先度**:全画面を同じフィデリティで?それとも重要な画面だけ詳細に?

## オプション

9. **HTMLモック**:静的HTMLで見た目を再現する必要がある?(レビュー効率向上のため)

10. **アクセシビリティ**:特に重視する項目(コントラスト/キーボード操作/スクリーンリーダー)?
