# 中期スパン手法（映画1本 / 1巻 / シーズン1本）

## 5) 三幕構成（Syd Field系）

**目的**：骨格の安定（起承転結が崩れない）

### 必要な成果物（最低4つ）

- Beginning（始まりの状態）
- End（終わりの状態）
- Plot Point I（Act2に押し込む反転）
- Plot Point II（Act3に押し込む反転）

### 手順

1. 終わりを1段落で固定（End）
2. 始まりを1段落で固定（Beginning）
3. PP1：主人公が「後戻り不能」になる出来事を決める
4. PP2：最終決戦に入らざるを得ない出来事を決める
5. Act2は「障害の連鎖」と「中盤の反転（Midpoint）」で2分割する

### チェック

- PP1で"問題が始まった"ではなく**人生が変わった**になってるか
- PP2で"頑張る気になった"ではなく**決着に入る**になってるか

## 6) 8シークエンス法

**目的**：中だるみ対策 / 塊ごとの差し替え耐性

### 典型配置

- Act1：S1・S2
- Act2：S3・S4・S5・S6
- Act3：S7・S8

### 手順

1. まずS4（中盤反転の前）とS6（どん底前）を決める
2. 各シークエンスに「ミニ目標」を置く
3. 各シークエンス末尾に「状況が変わる決着（Turn）」を置く
4. 同じ種類のシークエンスが連続してないかチェック

### テンプレート（1行＝1シークエンス）

```markdown
| S# | 状況(冒頭) | ミニ目標 | 障害 | 決着(反転) | 次の塊に生まれる新しい問題 |
|---|---|---|---|---|---|
```

## 7) Save the Cat!（15ビート）

**目的**：必要な節の抜け漏れ防止 / ピッチ・企画開発に強い

### 15ビート

Opening Image / Theme Stated / Set-Up / Catalyst / Debate / Break into Two / B Story / Fun and Games / Midpoint / Bad Guys Close In / All Is Lost / Dark Night of the Soul / Break into Three / Finale / Final Image

### 運用手順

1. まず **Catalyst / Break into Two / Midpoint / All Is Lost / Finale** だけ埋める
2. Theme Stated と Final Image を対応させる（最初と最後の対比）
3. Fun and Games は「この作品ならではの快感」を列挙して選ぶ
4. Bad Guys Close In を"外的圧力"と"内的欠点"の両方にする

### テンプレート

```markdown
| Beat名 | 何が起きる | 何が変わる(状況/関係/情報) | 次のビートの条件 |
|---|---|---|---|
```

## 8) Michael Hauge 6ステージ（＋5 Turning Points）

**目的**：外的ゴールと内的変化（恐れ→勇気）を同期させやすい

### 手順

1. 主人公の「恐れ（fear）」を明文化
2. 外的ゴール（欲しいもの）を1つに固定
3. 5つの転換点に「決断」を置く（出来事ではなく"選ぶ"）
4. 各ステージに「外的前進」と「内的抵抗」を必ず1つずつ入れる

### テンプレート

```markdown
| Stage# | 外的状況 | 外的ゴール進捗 | 内的状態(恐れ/防衛) | 決断 | 次ステージ条件 |
|---|---|---|---|---|---|
```

## 9) Truby 22ステップ

**目的**：複雑系（群像・政治・ミステリ）の材料チェックリスト

### 使い方

- 22を順番通りに埋めるより、**抜けがないか点検**に使う
- 特に序盤で必ず点検する項目：
  - 主人公の弱点・必要（Need）
  - 欲望（Desire）
  - 対立（Opponent / Mystery）
  - 世界（Story World）
  - 道徳論（Moral Argument）

### テンプレート

```markdown
| ブロック名 | 作品での具体(1〜3行) | どこで示す(章/シーン) | 未確定/確定 |
|---|---|---|---|
```

## 10) McKee：価値の反転（Value Shift）

**目的**：シーンの強度を上げる / 動いてない場面を消す

### 実装ルール

- **各シーンで価値が変わる**（例：信頼→不信、自由→拘束、安全→危険）
- 変わらないシーンは説明であってドラマじゃない可能性が高い

### テンプレート（シーン品質チェック）

```markdown
| シーンID | 主価値 Before→After | 主人公の目的 | 障害 | 決断 | 新情報 | 次シーンへの圧 |
|---|---|---|---|---|---|---|
```
