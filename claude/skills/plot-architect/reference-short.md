# 短期スパン手法（章 / シーン / 日々の執筆運用）

## 11) シーンカード（Index Cards）& ビートボード

**目的**：並べ替え最強 / リライト最強 / 構造が見える

### カード1枚に書くべき最小

- シーン見出し（場所・時間）
- 誰のシーンか（視点）
- 目的（Goal）
- 障害（Conflict）
- 反転（Turn）
- 最後の一撃（Button）

### 運用

1. まずカードで全シーンを作る（1枚＝1シーン）
2. Midpoint / All Is Lost / Finale などの"柱"カードを先に固定
3. 間を埋める
4. 余ったカードは削る（"かわいいけど不要"を切る）

---

## 12) Goal / Conflict / Turn / Button（シーンの最小単位）

**目的**：1シーンを必ず前進させる

### 定義

- **Goal**：主人公がこのシーンで達成したいこと
- **Conflict**：それを邪魔するもの
- **Turn**：結果、状況がどう変わったか
- **Button**：終わり際の一撃（次を見たくなる）

### チェック

- Goalが「会話する」になってないか（会話は手段）
- Conflictが弱いなら、時間制限・第三者・誤解・代償を足す
- Turnがないなら、シーンを統合するか削る

---

## 13) 章×筋マトリクス（JKR型）

**目的**：サブプロットの放置と詰め込みを防ぐ

### 作り方

- 行：Chapter 1..N
- 列：メイン筋 + サブ筋2〜4本
- セル：その章でその筋がどう前進するか（1行）

### 運用ルール

- "最低1筋は前進"
- 3章連続で空欄の筋があるなら、意図を説明できる必要がある

---

## 14) ステップアウトライン（Step Outline）

**目的**：シーン見出しの全列挙で全体を俯瞰する

### テンプレート（1行＝1シーン）

```markdown
| # | INT/EXT 場所 時間 | 登場人物 | 目的 | 障害 | 結果(反転) | 伏線ID |
|---|---|---|---|---|---|---|
```

### 運用

- まずステップアウトラインだけ完成させる
- 順に脚本化する（"良いシーンだけ"はやらない → 全体崩壊を防ぐ）

---

## 15) スノーフレーク法（Snowflake Method）

**目的**：ゼロから長編を段階的に作る（設計過多を防ぎつつ構造を作る）

### 手順

1. **1文サマリー**（ログライン）
2. **1段落サマリー**（大転換点 + 結末まで）
3. **主要キャラの1ページ要約**（欲望 / 恐れ / 変化）
4. **1ページあらすじ**（各文を段落に拡張）
5. **シーンリスト**（ステップアウトラインへ）

### 運用のコツ

- "次に行けない"ときは一段階戻る（詳細化しすぎを防ぐ）
- 1段落サマリーで「終わり」を必ず書く（永遠に決まらない病を防ぐ）

---

# 推奨フロー（破綻しにくい順番）

1. **長期**：シリーズ・ログライン → バイブル骨子 → タイムラインA/B → 伏線台帳
2. **中期**：8シークエンス → 三幕 or STCで節チェック → Haugeで内面同期 →（必要ならTruby点検）
3. **短期**：ステップアウトライン → シーンカード → 各シーンをG/C/T/B + 価値反転で磨く
