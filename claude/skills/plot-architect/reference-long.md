# 長期スパン手法（シリーズ / 複数巻 / TVシーズン）

## 1) シリーズ・ログライン（North Star）

**目的**：作品が長期化しても「何の話か」を失わない

### 手順

1. 主人公を"属性"ではなく"欠落（弱点）"で定義する
2. 作品全体で追う「外的ゴール」を1つに絞る
3. 対立軸（敵・制度・世界法則）を1つに絞る
4. 「失敗したら何を失う？」を1フレーズで書く
5. 1文に圧縮して、**30秒で説明できる**まで削る

### テンプレート

```text
欠落を抱えた[主人公]が、[外的ゴール]を目指すが、[対立軸]に阻まれ、
失敗すれば[賭け金]を失う。最終的に[変化]を遂げる物語。
```

---

## 2) シリーズ・バイブル（Show Bible）

**目的**：長期運用の設定・キャラ・テーマの台帳。チーム共有にも強い。

### 最低限の章立て

| セクション | 内容 |
|---|---|
| 作品概要 | ログライン + トーン |
| テーマ | 作品が言いたいこと / 問い |
| 世界観ルール | 魔法・技術・法律・制約 |
| キャラクター | 欲望 / 恐れ / 変化 / 関係性 |
| シリーズ大筋 | 始点 → 中盤 → 終点 |
| 各巻概要 | 1〜2行 × N |
| 禁止事項 | やらないこと（世界観矛盾・NG描写） |

### 運用ルール

- 「確定」「仮」「保留」をタグで分ける
- 例外が出たら必ず追記する

---

## 3) タイムライン法（客観時系列 / 提示順）

**目的**：矛盾ゼロ / 伏線回収漏れゼロ

### 手順

1. 大事件（世界が不可逆に変わるイベント）を箇条書き
2. 「客観時系列」に並べる（A）
3. 読者に見せる順に並べ替える（B）
4. AとBの差分（隠す情報・誤誘導・後出し）をメモ
5. 各情報に「証拠（根拠になるシーン）」を紐づける

### テンプレート（1行＝1イベント）

```markdown
| 日時/時期 | 場所 | 何が起きた | 誰が知ってる | 何が変わった | 根拠シーンID |
|---|---|---|---|---|---|
```

---

## 4) 伏線台帳（Clue / Red herring / Payoff）

**目的**：回収漏れ・回収弱さを潰す

### 手順

1. 伏線にIDを付ける（F001…）
2. **置く（Plant）**と**回収（Payoff）**を必ずペアで書く
3. 種類をタグ化：clue / red herring / setup / rule
4. 「回収の証明方法」を書く（台詞/映像/行動）

### テンプレート

```markdown
| ID | 種類 | 置く(巻/章) | 回収(巻/章) | 形(台詞/小物/描写) | 誤解誘導 | 回収の証明 |
|---|---|---|---|---|---|---|
```
