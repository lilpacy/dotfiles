# Vocabulary Coach (語彙コーチ) — Claude Code Skill

## TL;DR
単発の質問は「問いの枠」に回答が閉じやすい。そこで **回答フェーズ**の前に、専門家がやっている **語彙供給フェーズ**（探索）を分離して実行する。

本スキルは、あなたの課題を読んで
1) 枠外を含む関連語彙を増やし（NE / 探索）
2) 使える語彙へ絞り（TI / 評価）
3) 次の調査クエリへ落とす
ことに特化する。

---

## 目的
- 「もっとこういう語彙で調べると良い」をAIにやらせる
- 問いの枠を越えるための **言い換え** と **隣接領域** を意図的に混ぜる
- そのまま検索・読書・実装調査に使える **クエリ** を吐かせる

## 想定ユースケース
- 技術選定、UX改善、IA設計、顧客開発、運用設計などで「調べる入口」が見つからない
- すでに1〜2語はあるが、その周辺語彙（別名/略語/代表実装/手法名）が欲しい
- 課題が抽象的で、フレーム（見方）を変えたい

## 入力フォーマット（推奨）
最小:
- 課題: 1〜2段落

より良い:
- 目的（何を良くしたいか）
- 制約（時間・技術・組織・品質・コスト）
- 前提（現状の実装/運用/顧客）
- 既知のキーワード（すでに知ってる語）

## 出力フォーマット（スキルの規格）
1. **用語バンク（30語）**
   - 5観点（技術/UX/プロダクト/運用/ビジネス）×各6語
   - 各語に「一言定義」「効く理由」「検索クエリ例（JP+EN）」「別名/類義語」
2. **枠外リフレーム（3つ）**
   - 別分野のフレーム名
   - そのフレームで重要になる用語10個
   - 見える論点
3. **逆引き（任意）**
   - 欲しい効果→コア概念→実務用語→落とし穴
4. **次に検索すべきTop5〜10（JP+EN）**

---

## ワークフロー（NE→TI）
### Phase 1: NE（探索）
- 目的: 分布を広げる。網羅性>正確性。
- ルール:
  - 用語は「概念」「手法」「実装名」「評価指標」「アンチパターン」「フレーム名」を混ぜる
  - 必ず **別名・略語・英語表現** を付ける
  - 隣接領域（例: 情報検索、認知心理、在庫管理、分散システム、行動経済学）を混ぜる

### Phase 2: TI（評価）
- 目的: 使える語彙に絞る。実用性>網羅性。
- ルール:
  - 「いまの制約」と「次にやる行動」に繋がる語を優先
  - 最後に Top5〜10 クエリへ圧縮

---

## 失敗しやすいパターンと対策
- 失敗: 既知の語彙を言い換えただけで終わる
  - 対策: “隣接領域フレーム” を必ず3つ出す

- 失敗: 用語が多すぎて使えない
  - 対策: 末尾に Top5〜10 を必ず付ける（JP+EN）

- 失敗: 課題が薄くて的外れ
  - 対策: それでも仮説として出力しつつ、追加で聞くべき情報を3つだけ提示

---

## 品質チェック（自己採点）
- [ ] 30語が5観点に分かれている
- [ ] 各語に定義・理由・クエリ（JP+EN）・別名がある
- [ ] 3つの別フレームがあり、各10語ある
- [ ] 最後にTop5〜10の検索クエリへ圧縮されている
