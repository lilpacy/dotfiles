# 画面遷移図:最小インプット(これだけで作り始められる)

## 1) 粒度(どれで作る?)
- A 情報設計寄り / B 仕様寄り / C Wireflow寄り

## 2) 対象ユーザー(役割)
- 例:一般ユーザー / 管理者 / 編集者 / ゲスト

## 3) ゴール(何をもって完了?)
- 例:投稿を公開したら完了

## 4) 入口(Entry)
- 例:LP、ディープリンク、通知、ホーム

## 5) 画面候補(思いつく範囲で)
- 例:ログイン、ホーム、一覧、詳細、作成、設定、完了

## 6) 必須の分岐/例外(最低限)
- 未ログイン時の挙動
- 権限不足時の挙動
- 失敗(認証/保存/通信)
- 空状態(データなし)
- 戻る/キャンセル導線

## 7) 命名ルール(任意)
- ルート名 / Figmaフレーム名に合わせたい など
