# レビュー用チェックリスト

- [ ] 入口(Entry)が全て描かれている(LP/ディープリンク/通知など)
- [ ] ゴール(Goal)が明確(何をもって完了か)
- [ ] 主経路(Happy path)が最短で読める
- [ ] 戻り導線がある(戻る/キャンセル/閉じる)
- [ ] 例外系がある(失敗、権限不足、通信不可、空状態)
- [ ] 主要な分岐条件が矢印ラベルに書かれている(トリガー/条件/結果)
- [ ] 画面名が一貫している(実装ルート/Figmaと対応できる)
- [ ] 同一画面の状態増殖が過剰でない(注釈で表せている)
- [ ] スパゲッティ化していない(必要ならL0/L1/L2に分割できている)
- [ ] QA観点(失敗時の遷移/表示)が読み取れる
